# cli-doc-ai Documentation

## Overview

`cli-doc-ai` is a GitHub Action and CLI tool that automatically generates CHANGELOG entries using AI (OpenRouter) based on PR commits and diffs. It streamlines the documentation process by leveraging AI to summarize changes and create informative changelog entries.  It can be used both automatically within a GitHub workflow and manually via the command line.

## Architecture

The project consists of a core module (`src/core`) containing the logic for interacting with OpenRouter, analyzing git diffs, and generating changelog entries.  A CLI (`src/cli.ts`) provides a command-line interface for manual use, while `src/action.ts` serves as the entry point for the GitHub Action. The project utilizes external libraries like `@clack/prompts` for interactive prompts, `chalk` for styling, and `execa` for executing shell commands.

Data flow:

1.  **Input:**  PR commits/diffs (GitHub Action) or selected files/changes (CLI).
2.  **Analysis:** The `analyzer.ts` module scans the codebase and extracts relevant information.
3.  **Prompt Generation:**  `prompts.ts` creates a prompt for the AI model based on the extracted information.
4.  **AI Interaction:** `openrouter.ts` sends the prompt to OpenRouter and receives a generated summary.
5.  **Changelog Update:** `changelog.ts` updates the `CHANGELOG.md` file with the generated entry.
6.  **Output:** Updated `CHANGELOG.md` file.

## Key Components

*   **`src/action.ts`**:  Entry point for the GitHub Action. Orchestrates the entire process, including collecting PR metadata, calling the AI model, and updating the CHANGELOG.
*   **`src/cli.ts`**:  Provides the command-line interface for manual use.  Handles user interaction, configuration, and execution of the changelog generation process.
*   **`src/core/openrouter.ts`**:  Handles communication with the OpenRouter API.  Includes functions for sending prompts, handling API responses, and managing API keys.
*   **`src/core/changelog.ts`**:  Manages the `CHANGELOG.md` file.  Includes functions for reading, updating, and writing the changelog.
*   **`src/core/analyzer.ts`**: Scans the codebase to extract relevant information for the AI prompt, including file paths, content, and diff statistics.
*   **`src/core/git.ts`**:  Provides utility functions for interacting with Git, such as collecting commit messages and diffs.
*   **`src/core/prompts.ts`**: Defines the prompts sent to the AI model, tailored for different scenarios (PR summaries, release notes, single file documentation).
*   **`src/core/version.ts`**: Detects the project version from common manifest files (package.json, pyproject.toml, Cargo.toml) or git tags.
*   **`action.yml`**: Defines the GitHub Action metadata, inputs, and outputs.

## Data Models

*   **`PRContext`**:  Interface representing the context of a Pull Request, including title, author, commits, diffs, and repository name. Used as input for the AI prompts.
*   **`OpenRouterOptions`**: Interface for configuring the OpenRouter API call, including API key, model, and maximum tokens.
*   **`CodebaseSnapshot`**: Represents a snapshot of the codebase, including a directory tree and a list of scanned files.
*   **`ScannedFile`**: Represents a single file scanned by the analyzer, including its path, content, and number of lines.
*   **`ChangelogEntry`**: Represents a single entry in the CHANGELOG, including version, date, PR number, title, author, and summary.

## API / Interfaces

The project primarily exposes functionality through its GitHub Action and CLI.

**GitHub Action Inputs:**

*   `openrouter-api-key` (required): Your OpenRouter API key.
*   `openrouter-model` (optional): The OpenRouter model to use (default: `mistralai/mistral-7b-instruct:free`).
*   `changelog-path` (optional): The path to the CHANGELOG file (default: `CHANGELOG.md`).

**GitHub Action Outputs:**

*   `changelog-updated`:  `true` if the CHANGELOG file was updated, `false` otherwise.
*   `entry-version`: The version string used in the generated entry.

## Configuration

The project can be configured using environment variables and optional inputs.

*   **`OPENROUTER_API_KEY`**: (Required) Your OpenRouter API key.  Can be set in the environment or provided as an input to the GitHub Action.
*   **`OPENROUTER_MODEL`**: (Optional) The OpenRouter model to use.  Defaults to `mistralai/mistral-7b-instruct:free`.
*   **`CHANGELOG_PATH`**: (Optional) The path to the CHANGELOG file. Defaults to `CHANGELOG.md`.

Configuration is typically done via a `.env` file for local development and through GitHub Secrets for the GitHub Action.

## How to Run

**1. Local CLI Usage:**

*   **Install:** `bun add -g cli-doc-ai`
*   **Run:**
    *   With API key from environment: `OPENROUTER_API_KEY=sk-or-... doc-ai`
    *   Interactively: `doc-ai` (the CLI will prompt for the API key)

**2. GitHub Action Usage:**

*   Add the `OPENROUTER_API_KEY` secret to your GitHub repository (Settings -> Secrets and variables -> Actions).
*   Create a `.github/workflows/auto-docs.yml` file with the content from the `examples/caller-workflow.yml` file, replacing `YOUR_GITHUB_USER` with your GitHub username or organization.
*   The action will automatically run on every pull request opened against the `main` branch.
