# Release 1.1.0 — 2026-06-30

### New Features

*   New "Document business rules" mode (`docs/business-rules.md`): scans the codebase and extracts domain validations, invariants, and rules, grouped by module and pointing to where each one is enforced — meant to give an AI code-review agent (or a human reviewer) concrete criteria to check a PR against.
*   New "Document implementation templates" mode (`docs/templates.md`): extracts recurring conventions and "how to add a new X" recipes from the codebase, so new implementations and their reviews can be checked against an existing pattern instead of being judged from scratch.
*   New "Document a casual overview" mode (`docs/overview.md`): plain-language, jargon-free project summary aimed at onboarding and non-technical stakeholders.
*   New "Enrich a task description" mode: turns a rough task (pasted text, a local file, or a fetched GitHub issue) into a detailed implementation spec — goal, acceptance criteria, relevant business rules, suggested approach, edge cases, and open questions — using the documentation above as ground truth instead of guessing.

### Improvements

*   Extracted the shared "scan codebase → call AI → preview → write file" flow into a single `runDocAgent` helper, reused by all four codebase-scan documentation modes.

### Notes

*   These new documentation modes are CLI-only for now; the GitHub Action still only handles automatic CHANGELOG generation on PRs.
*   The "Enrich a task description" mode works best after running the business rules and templates modes at least once.
